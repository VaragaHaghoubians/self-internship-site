Cloud Coding Setup

Use this file when you want the 12-month self-internship to run like a real coding workspace.

Recommended beginner setup

Use GitHub Codespaces first.

Why:

  • It opens VS Code in the browser.
  • It gives you a terminal.
  • It can use the .devcontainer setup in this repository.
  • It lets you commit every task so your work is saved.
  • It is easier for a beginner than configuring a full local machine.

Step-by-step

  1. Create a GitHub account.
  2. Create a new repository named self-internship.
  3. Upload or push this whole folder to that repository.
  4. Open the repository in GitHub.
  5. Choose Code.
  6. Choose Codespaces.
  7. Choose Create codespace.
  8. Wait until VS Code opens in the browser.
  9. Open the terminal.
  10. Run:
python --version
node --version
git --version

If those commands work, your cloud coding workspace is ready.

Daily workflow

  1. Open self-internship-site/index.html in the website.
  2. Choose the current month.
  3. Read the task and expected output.
  4. Open the matching folder in Codespaces.
  5. Run the code in the terminal.
  6. Fix errors until the output matches the website.
  7. Paste the evidence into the website submission box.
  8. Commit your work:
git status
git add .
git commit -m "Complete month X task Y"
git push

When to use GitLab

GitLab is also good, especially if you want GitLab issues, merge requests, CI/CD, and GitLab Workspaces.

For a beginner, start with GitHub Codespaces unless you specifically want GitLab.

Mentor rule

Every task should produce evidence:

  • working code
  • terminal output
  • saved file
  • screenshot path
  • short explanation of what you learned
  • Git commit

If a task does not produce evidence, it is not finished yet.
