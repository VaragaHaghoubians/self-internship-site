Emergency Learning Workspace

This folder is your execution environment for a 2-month emergency study plan.

  • Month 1: Python, data science, machine learning, deep learning basics, SQL, MongoDB, Git, Linux
  • Month 2: Java

The goal is not to consume content. The goal is to write code every day, review your mistakes, and rebuild small things yourself.

What Is Already Set Up

This workspace already has:

  • a local virtual environment at .venv
  • core Python packages installed from requirements-core.txt
  • a Jupyter kernel named Python (emergency-path)
  • a local Git repository initialized in this folder
  • a folder structure for exercises, notebooks, projects, and progress tracking
  • an 8-week written roadmap
  • a curated GitHub links file for projects and beginner exercise repositories

What You Should Do First

Follow this order exactly:

  1. Open start-here.md.
  2. Open day-plans/index.md.
  3. Start with day-plans/week-01.md.
  4. Do Day 1 only.
  5. Put your code in practice/python/day-01/.
  6. Log mistakes in progress/mistakes.md.
  7. Log your daily summary in progress/daily-log.md.
  8. Commit your work.

Exact Commands To Use

Open PowerShell in this folder and run:

.\scripts\enter.ps1
python scripts\verify_env.py

If you want notebooks:

.\scripts\start-jupyter.ps1

If you ever need to reinstall packages:

.\.venv\Scripts\Activate.ps1
python -m pip install --upgrade pip
pip install -r requirements-core.txt

Optional deep learning packages:

pip install -r requirements-dl.txt

How To Study A Repo Properly

Use this process every time:

  1. Clone only one repository at a time.
  2. Read the README first.
  3. Run the code as-is.
  4. Read one small file or one lesson only.
  5. Rebuild the same thing in your own folder without copying line by line.
  6. Compare your version with the original.
  7. Record your mistakes in progress/mistakes.md.

Do not download many repos and skim them. That wastes time.

Daily Rules

  1. Activate the environment before coding.
  2. Put each exercise in the correct practice/ folder.
  3. Put notebooks in notebooks/.
  4. Put mini projects in projects/.
  5. Update progress/mistakes.md whenever something breaks.
  6. Update progress/daily-log.md at the end of the day.
  7. Commit at least once per day.
  8. Do not move to the next week unless you finish the current week's deliverables.
