Week 1: Terminal Commands Cheatsheet

What is the Terminal?

The terminal (also called "command line" or "shell") is a text-based way to control your computer. Instead of clicking icons, you type commands.

Why use it?

  • It's faster for repetitive tasks
  • Servers (where your code runs in the cloud) have NO screen — only a terminal
  • Every professional developer uses it daily

Setup: How to Open a Terminal on Windows

Option A: WSL 2 (Best — gives you real Linux)

  1. Open PowerShell as Administrator
  2. Type: wsl --install
  3. Restart your computer
  4. Open the "Ubuntu" app — you are now in Linux!

Option B: Git Bash (Quick, works now)

  1. Download Git from https://git-scm.com/download/win
  2. During install, choose "Git Bash Here" option
  3. Right-click any folder → "Git Bash Here"

🎯 YOUR TASK FOR WEEK 1

Open your terminal and run these commands ONE BY ONE. Do NOT copy-paste all at once. Type each one manually to build muscle memory.

# Step 1: Go to your home folder
cd ~

# Step 2: See what's already there
ls

# Step 3: Create your main lab folder
mkdir lab

# Step 4: Go into it
cd lab

# Step 5: Create three subfolders at once
mkdir projects datasets notes

# Step 6: Check that they were created
ls

# Step 7: Create a notes file
touch notes/week1-notes.txt

# Step 8: Go into notes and verify the file exists
cd notes
ls

# Step 9: Write something into the file (the > symbol means "write to file")
echo "I learned terminal commands today!" > week1-notes.txt

# Step 10: Read the file
cat week1-notes.txt

# Step 11: Go back to lab folder
cd ..

# Step 12: See the full structure
ls -R

If you did all of that without errors — congratulations! You passed Week 1.
