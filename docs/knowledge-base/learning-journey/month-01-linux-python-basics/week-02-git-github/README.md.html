Week 2: Git & GitHub — Version Control

What is Git? Why Do You Need It?

Imagine you're writing your thesis. You save versions:

  • thesis_v1.docx
  • thesis_v2_final.docx
  • thesis_v3_FINAL_REAL.docx

Git does this automatically, but for CODE. Every time you save (commit), Git remembers the exact state of all your files. You can go back any time.

GitHub is the website where you store your Git projects online. Recruiters WILL look at your GitHub profile. It is your coding portfolio.

Setup Git (One Time Only)

# Tell Git your name and email (use the same email as GitHub)
git config --global user.name "Your Full Name"
git config --global user.email "your@email.com"

# Verify it saved
git config --list

The 3 Core Git Commands You'll Use Every Day

git add .          →  "Stage" all changed files (mark them to be saved)
git commit -m ""   →  Save a snapshot with a message describing what changed
git push           →  Upload your snapshot to GitHub

Think of it like this:

  • git add = Put items in your shopping cart
  • git commit = Confirm the purchase
  • git push = The items get delivered (to GitHub)

Creating Your First GitHub Repository

Step 1: On the GitHub website

  1. Go to https://github.com and log in
  2. Click the green "New" button
  3. Name it: linux-data-journey
  4. Check "Add a README file"
  5. Click "Create repository"

Step 2: On your computer (in the terminal)

# Go to your lab/projects folder
cd ~/lab/projects

# Download your repository from GitHub (replace YOUR_USERNAME)
git clone https://github.com/YOUR_USERNAME/linux-data-journey.git

# Go into the downloaded folder
cd linux-data-journey

# See what's inside
ls

Step 3: Make a change and push it

# Create a new file
echo "# My Linux & Data Science Journey" >> NOTES.md
echo "Today I learned Git!" >> NOTES.md

# Stage the file
git add .

# Save a snapshot (the message describes what you did)
git commit -m "Add my first notes file"

# Upload to GitHub
git push

Now refresh your GitHub page — you'll see the file online!

Git Command Reference

CommandWhat it does
git initStart tracking a folder with Git
git clone <url>Download a repo from GitHub
git statusSee which files changed
git add .Stage ALL changed files
git add filenameStage ONE specific file
git commit -m "msg"Save a snapshot
git pushUpload to GitHub
git pullDownload latest changes from GitHub
git logSee history of all commits
