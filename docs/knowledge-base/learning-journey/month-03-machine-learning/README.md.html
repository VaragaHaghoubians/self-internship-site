🟠 Month 3: Machine Learning & Docker

Theme: Train models that make predictions. Package your code so it runs anywhere. Pre-requisite: You can clean data with Pandas, write SQL, and push to GitHub.

Why This Month Matters

You already know how to analyze data. Now you'll make it predict the future. Machine learning is the core of "Data Scientist" and "ML Engineer" job titles.

Docker lets you package your code + all its dependencies into a single container that runs identically on any machine — laptop, server, or cloud.

Week-by-Week Plan

Week 9 — SSH & Linux Productivity (continues terminal skills)

📁 Folder: week-09-ssh-productivity/

  • Learn to connect to remote servers: ssh user@server-ip
  • Learn powerful search commands: grep, find, history
  • Switch your GitHub from password to SSH key (more secure)

Week 9 (extra) — Time Series Analysis

📁 Folder: week-09-time-series/ 📄 File: time_series_analysis.py

  • Understand trend, seasonality, and noise components
  • DatetimeIndex slicing, resampling (resample()), rolling windows
  • Decomposition with seasonal_decompose
  • Stationarity test (ADF) and differencing
  • ACF / PACF plots to choose ARIMA parameters
  • Three forecasting models: Holt-Winters, SARIMA, lag-feature Random Forest
  • Connect to your real ESP32 SQLite data (month-04)

Week 10 — Linear Regression

📁 Folder: week-10-linear-regression/ 📄 File: linear_regression.py

  • Predict a number (e.g., house price, temperature tomorrow)
  • Learn Train/Test split — why you can't train and test on the same data
  • Learn Mean Squared Error (MSE) to measure how wrong your model is

Week 11 — Classification & Model Evaluation

📁 Folder: week-11-classification/ 📄 Files: random_forest.py, model_evaluation.py

  • Predict a category (e.g., spam/not-spam, will it rain yes/no)
  • Random Forests: an ensemble of many decision trees
  • Learn Accuracy, Precision, Recall, F1-score — what they mean and when to use each
  • Confusion matrix anatomy (TP/TN/FP/FN), ROC curve, AUC, cross-validation

Week 12 — Docker Containerization

📁 Folder: week-12-docker/ 📄 File: Dockerfile + ml_script.py

  • What is a Docker image vs container?
  • Write a Dockerfile to package your ML script
  • Run your code in a container with one command

🏆 Portfolio Project 2

Name: Containerized ML Pipeline What you do:

  1. Train a Random Forest model on the California Housing dataset
  2. Evaluate it (print accuracy/MSE scores)
  3. Save it as model.pkl using joblib
  4. Write a Dockerfile that packages your script
  5. Push everything to GitHub as: housing-price-predictor

✅ End of Month 3 Checklist

  • [ ] Understand the difference between regression and classification
  • [ ] Can split data into train/test sets
  • [ ] Can train a LinearRegression and RandomForestClassifier
  • [ ] Can calculate MSE, Accuracy, Precision, Recall
  • [ ] Can save a model with joblib
  • [ ] Can write a basic Dockerfile and run it
  • [ ] Portfolio Project 2 pushed to GitHub
