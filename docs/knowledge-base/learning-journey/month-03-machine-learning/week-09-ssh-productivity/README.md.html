Week 9: SSH & Linux Productivity

Why This Week Matters

You've been working on your own laptop so far. In real data science jobs, your code runs on remote servers — machines in the cloud with no screen, no mouse. The ONLY way to control them is through SSH.

This week you'll also learn the Linux commands that make professionals 10x faster.

Part 1: SSH — Connect to a Remote Server

SSH (Secure Shell) lets you open a terminal on a remote machine as if you were sitting in front of it.

# Basic SSH connection
ssh username@server-ip-address

# Example: connecting to an AWS EC2 server
ssh -i my-key.pem ubuntu@54.123.45.67

# Exit the remote session
exit

Set Up SSH Keys (more secure than passwords)

# Step 1: Generate your key pair (one time only — on YOUR computer)
ssh-keygen -t ed25519 -C "your_email@example.com"
# Press Enter to accept default file location (~/.ssh/id_ed25519)
# Optionally add a passphrase for extra security

# Step 2: View your PUBLIC key (this is safe to share)
cat ~/.ssh/id_ed25519.pub

# Step 3: Copy public key to a server
ssh-copy-id username@server-ip

# Step 4: From now on, connect without a password
ssh username@server-ip

Switch GitHub from HTTPS to SSH (recommended)

# Copy your public key
cat ~/.ssh/id_ed25519.pub
# → Go to GitHub → Settings → SSH Keys → New SSH Key → Paste it

# Test the connection
ssh -T git@github.com
# Should say: "Hi username! You've successfully authenticated..."

# Now clone/push without entering your password every time
git clone git@github.com:your-username/your-repo.git

Part 2: Power Search Commands

These three commands are used by every Linux professional every day.

grep — Search inside files

# Find all lines containing "error" in a log file
grep "error" server.log

# Case-insensitive search
grep -i "error" server.log

# Show line numbers
grep -n "error" server.log

# Search recursively through all files in a folder
grep -r "import pandas" ~/projects/

# Invert match — show lines that DON'T contain the word
grep -v "DEBUG" server.log

# Count matching lines
grep -c "error" server.log

# Combine with pipes:
# Show all Python files that import numpy
grep -rl "import numpy" ~/projects/

find — Search for files by name or type

# Find all .csv files in current folder and subfolders
find . -name "*.csv"

# Find all Python files
find . -name "*.py"

# Find files modified in the last 7 days
find . -mtime -7

# Find files larger than 100MB
find . -size +100M

# Find and delete all __pycache__ folders (useful for cleanup)
find . -type d -name "__pycache__" -exec rm -rf {} +

# Find all .log files and show their sizes
find . -name "*.log" -exec ls -lh {} \;

history — Reuse past commands

# See your last 50 commands
history | tail -50

# Search your history for a command you ran before
history | grep "docker"

# Re-run command number 42 from history
!42

# Re-run the last command
!!

# CTRL+R — interactive reverse search (most useful!)
# Press CTRL+R then type part of a command — it finds it instantly

Part 3: Other Productivity Commands

# --- PIPES & REDIRECTION ---
# | (pipe) sends output of one command to the next
ls -la | grep ".py"            # List only Python files
cat data.csv | head -20        # Show first 20 lines of a file
cat data.csv | wc -l           # Count lines in a file

# > redirect output to a file (overwrites)
ls > file_list.txt

# >> append output to a file
echo "new line" >> notes.txt

# --- SCREEN & TMUX (run jobs that survive logout) ---
# Without tmux: if you log out, your running script stops!
# With tmux: scripts keep running even after you disconnect

# Install tmux (Ubuntu)
sudo apt install tmux

# Start a new session named "training"
tmux new -s training

# Run your long script
python train_model.py

# Detach (leave it running): press CTRL+B then D
# Log out — script keeps running!

# Log back in later and re-attach
tmux attach -t training

# List all sessions
tmux ls

# --- PROCESS MANAGEMENT ---
# See what's running and how much CPU/memory it uses
top          # Live view (press Q to quit)
htop         # Better version of top (sudo apt install htop)

# See running Python processes
ps aux | grep python

# Kill a process by ID
kill 12345

# Force kill (if regular kill doesn't work)
kill -9 12345

# --- DISK & MEMORY ---
df -h          # Disk space (h = human readable: GB, MB)
du -sh *       # Size of each item in current folder
free -h        # RAM usage

🎯 YOUR TASKS FOR WEEK 9

Task 1: Set Up SSH Keys for GitHub

  1. Generate an SSH key pair: ssh-keygen -t ed25519 -C "your@email.com"
  2. Add the public key to your GitHub account
  3. Test: ssh -T git@github.com
  4. Clone one of your existing repos using the SSH URL (git@github.com:...)

Task 2: Search Practice

Create a file called practice_log.txt with these contents:

INFO: Server started
DEBUG: Loading config
ERROR: Database connection failed
INFO: Retrying connection
ERROR: Timeout after 30 seconds
DEBUG: Memory usage 45%
INFO: Connection restored

Then answer these questions using grep:

  • How many ERROR lines are there?
  • Show all lines that are NOT DEBUG
  • Search case-insensitively for "info"

Task 3: Find Your Python Files

Run this from your home folder:

find ~ -name "*.py" 2>/dev/null | head -20

How many Python files have you written so far?

Task 4: History Spelunking

Run history | grep "git" — what git commands have you used most?
