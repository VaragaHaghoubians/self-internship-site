// DAY 01 — WHAT IS ARDUINO? YOUR FIRST SKETCH
// ==============================================================================
//
// TEACHER SAYS:
// Arduino is a MICROCONTROLLER — a tiny computer with no operating system.
// It runs ONE program (called a sketch) forever, in a loop.
// Perfect for: reading sensors, controlling motors, LEDs, displays.
//
// ARDUINO vs RASPBERRY PI:
//   Arduino       → No OS. Runs one sketch. 5V logic. Great for hardware control.
//   Raspberry Pi  → Runs Linux. Runs Python. More powerful, more power hungry.
//   They often WORK TOGETHER: Pi handles data/internet, Arduino handles sensors/motors.
//
// ARDUINO IDE SETUP:
//   1. Download: https://www.arduino.cc/en/software
//   2. Connect Arduino via USB
//   3. Tools → Board → Arduino Uno (or your board)
//   4. Tools → Port → select the COM port that appeared
//   5. Click Upload (→ button) to upload the sketch
//
// EVERY ARDUINO SKETCH HAS TWO REQUIRED FUNCTIONS:
//   setup() → runs ONCE when the board powers on or resets
//   loop()  → runs FOREVER after setup() finishes (like a while True in Python)
//
// THE SIMPLEST SKETCH — "Hello, World!" for Arduino:
// (The built-in LED is connected to pin 13 on most Arduino boards)
//
// CIRCUIT: No wiring needed! Arduino has a built-in LED on pin 13.

// DAY 02 — DIGITAL OUTPUT: EXTERNAL LED
// ==============================================================================
//
// TEACHER SAYS:
// The built-in LED is convenient, but let's control an EXTERNAL LED.
// Digital pins can be HIGH (5V) or LOW (0V).
// Always use a RESISTOR with LEDs — without it, the LED will burn out!
//
// RESISTOR VALUE for LED:
//   R = (Vsupply - Vled) / Iled = (5V - 2V) / 0.02A = 150Ω
//   Use 220Ω to be safe (higher resistance = dimmer but safer).
//
// CIRCUIT WIRING:
//   Arduino Pin 8  → 220Ω resistor → LED (long leg/anode) → LED (short leg/cathode) → GND
//
// C++ BASICS FOR ARDUINO:
//   int, float, bool, char   → data types (same as Java)
//   const int PIN = 8;       → constant (won't change)
//   #define LED_PIN 8        → macro (replaced before compile, no type)
//   void functionName() {}   → function with no return value
//   int functionName() {}    → function returning int
//   // comment               → single line comment
//   /* comment */            → multi-line comment

// ==============================================================================
// DAY 05 — PWM: ANALOG OUTPUT (LED DIMMING)
// ==============================================================================
//
// TEACHER SAYS:
// Arduino digital pins are only ON or OFF — no in-between.
// PWM (Pulse Width Modulation) FAKES an analog output by switching very fast.
// Duty cycle: 0% = always OFF, 50% = half on half off, 100% = always ON.
// The LED averages the ON/OFF and appears dim or bright.
//
// analogWrite(pin, value)  → value 0 to 255
//   0   → always OFF (0%)
//   127 → 50% duty cycle (half brightness)
//   255 → always ON (100%)
//
// IMPORTANT: Only certain pins support PWM!
//   Arduino Uno PWM pins: 3, 5, 6, 9, 10, 11  (marked with ~ on the board)
//
// CIRCUIT: LED with 220Ω resistor on pin 9 (PWM pin)

const int PWM_LED_PIN = 9;    // Must be a PWM pin (~)
