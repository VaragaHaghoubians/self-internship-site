Part 1 — Linear Programming with PuLP

What is Linear Programming?

An LP problem has three components:

  1. Decision variables — what you control (e.g., how many units to ship on each route)
  2. Objective function — what you minimize or maximize (e.g., total cost)
  3. Constraints — the limits of the real world (e.g., warehouse capacity, truck capacity)

You already solved these by hand in your IE courses. PuLP just does the algebra for you.

Installation

pip install pulp

PuLP uses the free CBC solver by default. For larger problems, install HiGHS:

pip install highspy

Minimal Working Example: Production Planning

import pulp

# The Problem:
# A factory makes two products: A and B.
# Product A earns $25 profit, Product B earns $30 profit.
# Constraint: machine hours limited to 40/week
# Product A needs 2 hours, Product B needs 4 hours.
# Question: how many of each to produce to maximize profit?

prob = pulp.LpProblem("production_plan", pulp.LpMaximize)

# Decision variables (non-negative integers)
x_a = pulp.LpVariable("units_A", lowBound=0, cat="Integer")
x_b = pulp.LpVariable("units_B", lowBound=0, cat="Integer")

# Objective: maximize profit
prob += 25 * x_a + 30 * x_b

# Constraint: machine hours
prob += 2 * x_a + 4 * x_b <= 40

# Solve
status = prob.solve(pulp.PULP_CBC_CMD(msg=0))

print(f"Status: {pulp.LpStatus[status]}")
print(f"Units A: {x_a.value()}")
print(f"Units B: {x_b.value()}")
print(f"Max Profit: ${pulp.value(prob.objective)}")

Transportation Problem (Classic IE Problem)

import pulp

# Ship goods from 3 warehouses to 4 customers at minimum cost.
# supply[i] = capacity of warehouse i
# demand[j] = requirement of customer j
# cost[i][j] = cost per unit shipped from i to j

warehouses = ["W1", "W2", "W3"]
customers  = ["C1", "C2", "C3", "C4"]

supply = {"W1": 120, "W2": 80,  "W3": 80}
demand = {"C1": 150, "C2": 70,  "C3": 100, "C4": 60}  # note: demand < supply intentionally

cost = {
    ("W1","C1"): 2, ("W1","C2"): 3, ("W1","C3"): 1, ("W1","C4"): 5,
    ("W2","C1"): 7, ("W2","C2"): 3, ("W2","C3"): 4, ("W2","C4"): 6,
    ("W3","C1"): 4, ("W3","C2"): 6, ("W3","C3"): 4, ("W3","C4"): 3,
}

prob = pulp.LpProblem("transportation", pulp.LpMinimize)

# Decision variables: flow on each route
flow = {
    (i, j): pulp.LpVariable(f"flow_{i}_{j}", lowBound=0)
    for i in warehouses for j in customers
}

# Objective: minimize total shipping cost
prob += pulp.lpSum(cost[i, j] * flow[i, j] for i in warehouses for j in customers)

# Supply constraints: don't ship more than each warehouse has
for i in warehouses:
    prob += pulp.lpSum(flow[i, j] for j in customers) <= supply[i]

# Demand constraints: each customer must receive their full demand
for j in customers:
    prob += pulp.lpSum(flow[i, j] for i in warehouses) >= demand[j]

prob.solve(pulp.PULP_CBC_CMD(msg=0))

print("Optimal shipping plan:")
for (i, j), var in flow.items():
    if var.value() > 0:
        print(f"  {i} → {j}: {var.value():.0f} units (cost: ${cost[i,j] * var.value():.0f})")
print(f"Total cost: ${pulp.value(prob.objective):.0f}")

Part 2 — Google OR-Tools

OR-Tools is the industry standard for combinatorial optimization problems that LP cannot solve efficiently: vehicle routing, scheduling, assignment, bin packing.

Installation

pip install ortools

The Assignment Problem (Cold-Chain Capstone Core)

Given N inventory items and M warehouses, assign each item to a warehouse to minimize total relocation cost. This is what your cold-chain capstone optimizer does when a cooling failure is predicted.

from ortools.sat.python import cp_model

# Scenario: 3 pallets of inventory, 3 destination warehouses
# cost[pallet][warehouse] = estimated spoilage/transport cost in $
pallets     = ["insulin_pallet", "vaccine_pallet", "frozen_food_pallet"]
warehouses  = ["warehouse_A",    "warehouse_B",    "warehouse_C"]

cost = [
    [120, 200, 180],   # insulin pallet to A/B/C
    [150,  90, 210],   # vaccine pallet to A/B/C
    [180, 160,  80],   # frozen food pallet to A/B/C
]

# Warehouse capacities (max pallets they can absorb)
capacity = {"warehouse_A": 2, "warehouse_B": 1, "warehouse_C": 2}

model = cp_model.CpModel()

# Decision variables: assign[pallet][warehouse] is 1 if that pallet goes there
assign = {}
for p in range(len(pallets)):
    for w in range(len(warehouses)):
        assign[p, w] = model.NewBoolVar(f"assign_{p}_{w}")

# Constraint 1: each pallet must go to exactly one warehouse
for p in range(len(pallets)):
    model.AddExactlyOne(assign[p, w] for w in range(len(warehouses)))

# Constraint 2: warehouse capacity limits
cap_list = [capacity[w] for w in warehouses]
for w in range(len(warehouses)):
    model.Add(sum(assign[p, w] for p in range(len(pallets))) <= cap_list[w])

# Objective: minimize total relocation cost
model.Minimize(
    sum(cost[p][w] * assign[p, w] for p in range(len(pallets)) for w in range(len(warehouses)))
)

solver = cp_model.CpSolver()
status = solver.Solve(model)

if status in (cp_model.OPTIMAL, cp_model.FEASIBLE):
    print("Optimal inventory transfer plan:")
    for p in range(len(pallets)):
        for w in range(len(warehouses)):
            if solver.Value(assign[p, w]):
                print(f"  {pallets[p]} → {warehouses[w]}  (cost: ${cost[p][w]})")
    print(f"Total cost: ${solver.ObjectiveValue():.0f}")

Vehicle Routing Problem (VRP) — The Canonical Logistics Problem

The VRP asks: given N delivery locations and K trucks, find routes for each truck that minimize total distance while respecting capacity limits. This is what Amazon solves billions of times per day.

from ortools.constraint_solver import routing_enums_pb2
from ortools.constraint_solver import pywrapcp

def build_distance_matrix(locations: list[tuple]) -> list[list[int]]:
    """
    Computes integer distance matrix from (lat, lon) coordinates.
    OR-Tools requires integer costs, so multiply by 1000 and round.
    """
    import math
    n = len(locations)
    matrix = []
    for i in range(n):
        row = []
        for j in range(n):
            lat1, lon1 = locations[i]
            lat2, lon2 = locations[j]
            # Euclidean approximation — good enough for small regions
            dist = math.sqrt((lat2 - lat1)**2 + (lon2 - lon1)**2) * 111_000  # meters
            row.append(int(dist))
        matrix.append(row)
    return matrix


def solve_vrp(locations, demands, vehicle_capacities, depot_index=0):
    """
    Solve a capacitated VRP.

    Args:
        locations:         list of (lat, lon) tuples; index 0 is the depot
        demands:           units required at each location (0 for depot)
        vehicle_capacities: list of capacity per vehicle
        depot_index:       index of the depot in locations

    Returns:
        list of routes, where each route is a list of location indices
    """
    distance_matrix = build_distance_matrix(locations)
    n_vehicles = len(vehicle_capacities)

    manager = pywrapcp.RoutingIndexManager(len(locations), n_vehicles, depot_index)
    routing = pywrapcp.RoutingModel(manager)

    # Distance callback
    def distance_callback(from_idx, to_idx):
        from_node = manager.IndexToNode(from_idx)
        to_node   = manager.IndexToNode(to_idx)
        return distance_matrix[from_node][to_node]

    transit_cb_idx = routing.RegisterTransitCallback(distance_callback)
    routing.SetArcCostEvaluatorOfAllVehicles(transit_cb_idx)

    # Capacity constraint
    def demand_callback(from_idx):
        return demands[manager.IndexToNode(from_idx)]

    demand_cb_idx = routing.RegisterUnaryTransitCallback(demand_callback)
    routing.AddDimensionWithVehicleCapacity(
        demand_cb_idx,
        0,                   # null capacity slack
        vehicle_capacities,  # vehicle maximum capacities
        True,                # start cumul to zero
        "Capacity",
    )

    # Search parameters
    search_params = pywrapcp.DefaultRoutingSearchParameters()
    search_params.first_solution_strategy = (
        routing_enums_pb2.FirstSolutionStrategy.PATH_CHEAPEST_ARC
    )
    search_params.local_search_metaheuristic = (
        routing_enums_pb2.LocalSearchMetaheuristic.GUIDED_LOCAL_SEARCH
    )
    search_params.time_limit.seconds = 30

    solution = routing.SolveWithParameters(search_params)

    if not solution:
        return None

    routes = []
    for v in range(n_vehicles):
        route = []
        idx = routing.Start(v)
        while not routing.IsEnd(idx):
            route.append(manager.IndexToNode(idx))
            idx = solution.Value(routing.NextVar(idx))
        route.append(manager.IndexToNode(idx))  # append depot at end
        routes.append(route)

    return routes


# --- Example usage ---
if __name__ == "__main__":
    # Depot at index 0, then 4 delivery locations
    locations = [
        (40.712, -74.006),   # depot: New York
        (40.730, -73.935),   # stop 1
        (40.650, -73.950),   # stop 2
        (40.680, -74.044),   # stop 3
        (40.740, -74.178),   # stop 4
    ]
    demands           = [0, 10, 7, 13, 15]     # depot has 0 demand
    vehicle_capacities = [25, 20]               # 2 trucks

    routes = solve_vrp(locations, demands, vehicle_capacities)
    if routes:
        for i, route in enumerate(routes):
            print(f"Truck {i + 1}: {' → '.join(str(r) for r in route)}")

Key Concepts to Be Able to Explain in an Interview

ConceptWhat to say
Why LP?"LP finds the global optimum in polynomial time for continuous problems. It gives me a provably optimal answer, not just a heuristic."
Why OR-Tools over LP for routing?"Routing problems are combinatorial — the number of possible routes grows factorially. OR-Tools uses constraint propagation + heuristic search to find near-optimal solutions in bounded time."
How do you know your model is good?"I compare its output to the baseline (greedy nearest-neighbor for routing, or manual expert rules for assignment). If it consistently beats baseline by >15%, it earns its place in production."
What is the connection between your ML model and the optimizer?"The ML model quantifies the risk. The optimizer decides the response. Neither is useful without the other — that's the IE advantage."
