Phase 3: Full-Stack Web Development

Duration: Month 5–6 Core Goal: Build a complete, user-facing web platform around your data and ML models.

By the end of this phase you can design a Django web application with a real database, user authentication, and a background ML pipeline — the foundation of every modern data product.

Month 5 — Django Fundamentals

📁 month-05-django-fundamentals/

Skills to master:

  • Django MVT architecture: Models, Views, Templates
  • Django ORM: defining models, running migrations, querying the database
  • URL routing, forms, and class-based views
  • Django Admin: instant admin panel for your data models
  • User authentication: login, logout, registration

Deliverable: A CRUD web app — e.g., a project management tracker where users can create, read, update, and delete tasks. Runs locally and is version-controlled on GitHub.

Resources: month-05-django-fundamentals/resources/ (django_zero_to_hero.py)

Month 6 — Integrating ML Into Django

📁 month-06-ml-into-django/

Skills to master:

  • File uploads: accept CSV or image uploads via Django forms
  • Background tasks with Celery + Redis: run ML jobs without blocking the web request
  • Serving model predictions through Django views
  • Displaying results on a dashboard (charts with Chart.js or Plotly)
  • Environment variables with django-environ / .env files (security)

Deliverable: A web app where users:

  1. Upload a CSV file via a form
  2. A Celery background task runs an ML model on the data
  3. When done, the user sees an interactive results dashboard

Resources: month-06-ml-into-django/resources/ (ml_script.py) Reference: knowledge-base/zero-to-hero-guides/django/django_zero_to_hero.py
