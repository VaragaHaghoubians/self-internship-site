Goal: combine sensor time-series data with unstructured technician notes

class TechnicianNotesStore:
    """
    Embeds technician maintenance notes and stores them in a vector database
    for retrieval during report generation.

    What is RAG?
        Retrieval-Augmented Generation.
        Instead of relying on the LLM's internal (possibly wrong) knowledge,
        you retrieve relevant documents and inject them into the prompt.
        The LLM then reasons over documents it has never seen before.

    Three steps:
        1. Embed: convert each note to a dense vector (sentence-transformers).
        2. Store:  insert vectors into ChromaDB.
        3. Retrieve: embed a query, find the top-k nearest vectors.

    Synthetic data strategy:
        You don't have real technician notes. Generate them with an LLM.
        Prompt pattern:
            "Write a short technician maintenance note (2-4 sentences) for an
             aircraft engine showing these sensor patterns: {summary}.
             Include a probable cause and recommended action."
