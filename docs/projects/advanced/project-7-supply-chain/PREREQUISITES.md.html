Prerequisites — Project 7: Generative Supply Chain Optimizer

Role: Operations Data Scientist / AI Systems Engineer | Phase: Advanced (after Month 9) | Strongest IE superpower demo

This project combines three of your IE superpowers in one system: graph-based network modelling (NetworkX), demand forecasting (Prophet), and linear programming (PuLP) — then adds an LLM risk agent that monitors real-world disruption events and triggers the optimiser. This is the project that most directly answers the question: "Why should we hire an Industrial Engineer over a pure ML engineer?"

Part 1 — Supply Chain Network Modelling with NetworkX

What a supply chain graph is

Your supply chain is a directed graph: nodes are facilities (suppliers, warehouses, demand points), and edges are transport routes with associated costs and capacities.

Supplier A (Shanghai) ──cost: $1.20/unit──► Warehouse B (Rotterdam) ──► Customer C (Berlin)
Supplier A (Shanghai) ──cost: $0.90/unit──► Warehouse D (Singapore) ──► Customer E (Sydney)

NetworkX models this exactly:

import networkx as nx

G = nx.DiGraph()   # directed: flow goes one way

# Add facility nodes with attributes
G.add_node("Shanghai",   type="supplier",   capacity=10_000, lat=31.2, lon=121.5)
G.add_node("Rotterdam",  type="warehouse",  capacity=8_000,  lat=51.9, lon=4.5)
G.add_node("Berlin",     type="demand",     demand=3_000,    lat=52.5, lon=13.4)

# Add transport routes (edges) with cost and capacity
G.add_edge("Shanghai", "Rotterdam",  cost_per_unit=1.20, capacity=6_000, transit_days=22)
G.add_edge("Shanghai", "Singapore",  cost_per_unit=0.40, capacity=5_000, transit_days=4)
G.add_edge("Rotterdam", "Berlin",    cost_per_unit=0.08, capacity=4_000, transit_days=2)

Key NetworkX operations you will use

# Find the cheapest path (Dijkstra)
path = nx.dijkstra_path(G, source="Shanghai", target="Berlin", weight="cost_per_unit")
cost = nx.dijkstra_path_length(G, source="Shanghai", target="Berlin", weight="cost_per_unit")

# Find bottleneck nodes — removing these disconnects the network most
centrality = nx.betweenness_centrality(G, weight="cost_per_unit")
top_bottleneck = max(centrality, key=centrality.get)

# Remove a disrupted route (e.g., Suez Canal closure)
G.remove_edge("Shanghai", "Rotterdam")

# Check if the network is still connected after disruption
nx.is_weakly_connected(G)

# Find all alternative paths
all_paths = list(nx.all_simple_paths(G, source="Shanghai", target="Berlin"))

Part 2 — Demand Forecasting with Prophet

Prophet is Facebook's open-source time-series forecasting library. It is designed for business time-series data that has seasonal patterns and holidays.

from prophet import Prophet
import pandas as pd

# Prophet requires exactly two columns: 'ds' (datestamp) and 'y' (value)
df_prophet = demand_df.rename(columns={"date": "ds", "units_sold": "y"})

model = Prophet(
    yearly_seasonality=True,
    weekly_seasonality=True,
    daily_seasonality=False,   # supply chain data is usually weekly/monthly
    seasonality_mode="multiplicative",   # use when seasonality scales with trend
)

# Add country holidays if relevant
model.add_country_holidays(country_name="US")

model.fit(df_prophet)

# Forecast the next 12 weeks
future = model.make_future_dataframe(periods=12, freq="W")
forecast = model.predict(future)

# Key output columns:
# forecast["yhat"]       — point forecast
# forecast["yhat_lower"] — lower confidence interval
# forecast["yhat_upper"] — upper confidence interval

Why Prophet over ARIMA for supply chain? Prophet handles missing data, outliers, and multiple seasonalities automatically. It also separates trend, weekly seasonality, and yearly seasonality into interpretable components — useful for explaining forecasts to a supply chain manager.

Part 3 — Linear Programming with PuLP

The Facility Location Problem

"Given K candidate facilities, which ones to open, and how to route flow between them, to minimise total cost while satisfying all demand?" This is a classic LP/MIP problem.

from pulp import LpProblem, LpVariable, LpMinimize, lpSum, value, PULP_CBC_CMD

# Problem definition
prob = LpProblem("supply_chain_optimisation", LpMinimize)

# Decision variables
# x[i][j] = units shipped from facility i to demand node j
facilities = ["Shanghai", "Rotterdam", "Singapore"]
demand_nodes = ["Berlin", "Sydney", "New York"]

x = {
    (i, j): LpVariable(f"flow_{i}_{j}", lowBound=0)
    for i in facilities for j in demand_nodes
}
# y[i] = 1 if facility i is open, 0 otherwise
y = {i: LpVariable(f"open_{i}", cat="Binary") for i in facilities}

# Objective: minimise transport cost + facility fixed costs
transport_costs = {("Shanghai", "Berlin"): 1.28, ("Rotterdam", "Berlin"): 0.08, ...}
fixed_costs = {"Shanghai": 50_000, "Rotterdam": 30_000, "Singapore": 25_000}

prob += (
    lpSum(transport_costs.get((i, j), 0) * x[i, j] for i in facilities for j in demand_nodes)
    + lpSum(fixed_costs[i] * y[i] for i in facilities)
)

# Constraints: demand must be satisfied
demand = {"Berlin": 3_000, "Sydney": 2_500, "New York": 4_000}
for j in demand_nodes:
    prob += lpSum(x[i, j] for i in facilities) >= demand[j]

# Constraints: capacity cannot be exceeded at open facilities
capacity = {"Shanghai": 10_000, "Rotterdam": 8_000, "Singapore": 5_000}
for i in facilities:
    prob += lpSum(x[i, j] for j in demand_nodes) <= capacity[i] * y[i]

# Solve
prob.solve(PULP_CBC_CMD(msg=0))

# Read results
for i in facilities:
    if value(y[i]) > 0.5:
        print(f"Open: {i}")
for (i, j), var in x.items():
    if value(var) > 0:
        print(f"  Ship {value(var):.0f} units from {i} to {j}")

Three solution variants to generate:

  1. Cost-optimal: minimise total cost only
  2. Resilience-optimal: maximise number of open routes (spread risk, ignore cost)
  3. Balanced: cost + resilience with equal weighting — a Pareto frontier point

Present all three to the user via the dashboard with a side-by-side ROI comparison table. The human picks the winner. This is the human-in-the-loop design pattern.

Part 4 — LLM Risk Agent

The Risk Agent monitors external disruption events (RSS feeds, news APIs) and decides which routes to disable in the graph.

import feedparser

def fetch_disruption_events(rss_urls: list[str]) -> list[str]:
    """
    Fetches recent news headlines from RSS feeds.
    Each headline is a potential supply chain disruption event.
    """
    headlines = []
    for url in rss_urls:
        feed = feedparser.parse(url)
        for entry in feed.entries[:5]:  # most recent 5 headlines
            headlines.append(entry.title)
    return headlines

# Example RSS feeds relevant to supply chain:
RSS_FEEDS = [
    "https://feeds.reuters.com/reuters/businessNews",
    "https://www.supplychainbrain.com/rss/articles",
]

The LLM then assesses each headline and decides which nodes/edges to disable:

import openai, json

def assess_disruption(headlines: list[str], network_nodes: list[str]) -> dict:
    prompt = f"""
You are a supply chain risk analyst. Based on the following news headlines,
identify which of these supply chain nodes are at risk of disruption.
Return JSON: {{"affected_nodes": [...], "reason": "..."}}

Headlines:
{chr(10).join(f"- {h}" for h in headlines)}

Network nodes: {network_nodes}
"""
    response = openai.chat.completions.create(
        model="gpt-4o-mini",
        messages=[{"role": "user", "content": prompt}],
        response_format={"type": "json_object"},
    )
    return json.loads(response.choices[0].message.content)

Map Visualisation with Plotly

The dashboard should show the supply chain network on an actual map. This is the demo moment — seeing freight routes drawn on a world map is far more compelling than a table.

import plotly.graph_objects as go

def plot_supply_chain(G: nx.DiGraph) -> go.Figure:
    fig = go.Figure()

    # Draw edges (routes)
    for u, v, data in G.edges(data=True):
        lat_u = G.nodes[u]["lat"]; lon_u = G.nodes[u]["lon"]
        lat_v = G.nodes[v]["lat"]; lon_v = G.nodes[v]["lon"]
        fig.add_trace(go.Scattergeo(
            lat=[lat_u, lat_v, None],
            lon=[lon_u, lon_v, None],
            mode="lines",
            line=dict(width=1, color="blue"),
            showlegend=False,
        ))

    # Draw nodes (facilities)
    for node, data in G.nodes(data=True):
        colour = {"supplier": "green", "warehouse": "orange", "demand": "red"}.get(data.get("type"), "grey")
        fig.add_trace(go.Scattergeo(
            lat=[data["lat"]], lon=[data["lon"]],
            mode="markers+text",
            marker=dict(size=10, color=colour),
            text=[node],
            textposition="top center",
        ))

    fig.update_layout(
        geo=dict(showland=True, landcolor="lightgrey"),
        title="Supply Chain Network",
        height=500,
    )
    return fig

Operations Research Concepts to Understand

Before coding the PuLP section, understand these concepts — they are what differentiates you from a pure ML engineer:

ConceptWhat it meansWhere it appears
Decision variablesThe unknowns the solver choosesx[i][j] = units to ship
Objective functionWhat to minimise or maximiseMinimise total transport + fixed cost
ConstraintsRules the solution must satisfyDemand must be met; capacity cannot be exceeded
Binary variables0 or 1 — "open or closed"y[i] = whether to open facility i
MIP (Mixed Integer Programming)LP with some integer/binary variablesRequired for facility location (you can't open 0.7 of a warehouse)
Feasibility vs. optimalityA feasible solution satisfies constraints; an optimal one minimises the objectiveYour solver may return "infeasible" — debug constraints first

The interview question: "Walk me through the LP formulation for this." You should be able to write the objective function and constraints on a whiteboard without code.

What You Do NOT Need for This Project

  • SimPy (no simulation — the network is modelled as a static graph)
  • BERT or transformer models (the LLM is called via API, not fine-tuned)
  • IoT hardware (supply chain data comes from CSV files or APIs)
  • Survival analysis (that is Project 6)
