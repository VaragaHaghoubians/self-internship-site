"""
PROJECT 3 — Generative Supply Chain Optimizer
===============================================
Goal: a supply chain network that detects external disruption events
and autonomously generates multiple redesign candidates with explicit
cost vs. resilience trade-offs. Human picks the winner.

This is the "Industry 5.0" concept in code:
    resilience + human-in-the-loop decision-making.

Pipeline:
    Network graph (NetworkX)
        → demand forecaster (Prophet)
        → Risk Agent (monitors events, disables routes)
        → Network Optimizer (PuLP linear program)
        → three redesign candidates (cost / balanced / resilience)
        → Streamlit map dashboard with ROI comparison table

Suggested libraries (research before importing):
    networkx          — directed graph for the supply chain
    prophet           — time-series demand forecasting
    pulp              — linear programming optimizer
    plotly / folium   — interactive geographic map
    feedparser        — RSS feed parsing for real news events
    openai / anthropic — LLM for risk assessment and reasoning
    streamlit         — dashboard

Dataset options for demand history:
    - Rossmann Store Sales (Kaggle) — retail demand, easy to adapt
    - M5 Forecasting Competition (Kaggle) — hierarchical, realistic
    - Or generate synthetic demand with trend + seasonality + noise

Key concept to understand before coding:
    A "facility location problem" asks: given K candidate sites,
    which subset to open, and how to route flow, to minimize total cost
    while satisfying all demand?
    This is NP-hard in general, but a small instance (5-20 nodes) solves
    in seconds with a linear programming relaxation using PuLP.
"""

class NetworkOptimizer:
    """
    Formulates and solves a facility location problem using PuLP.

    The optimization problem:
        Decision variables:
            x_i ∈ {0, 1}  — open candidate warehouse i (1) or not (0)
            f_ij ≥ 0       — units of flow on edge (i → j)

        Objective (minimize):
            Σ fixed_cost_i * x_i          (warehouse opening costs)
          + Σ cost_per_unit_ij * f_ij     (transportation costs)
          - weight_resilience * resilience_proxy   (reward resilience)

        Constraints:
            Σ f_ij (inbound to demand j) ≥ demand_j   for all demand nodes j
            Σ f_ij (outbound from i)     ≤ capacity_i * x_i   for all warehouses i
            f_ij ≥ 0

    Resilience proxy in the objective:
        You can't directly optimize edge connectivity (non-linear).
        Instead, penalize single-source dependency:
        resilience_proxy = number of demand nodes with ≥ 2 active inbound routes.
        This is a linear function of x_i and route choices — include it.

          It covers exactly this pattern: decision variables, objective, constraints, solve.
