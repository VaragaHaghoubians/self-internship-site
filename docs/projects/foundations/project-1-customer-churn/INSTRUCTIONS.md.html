Phase 5 — Statistical Hypothesis Testing (1 hour)

Why this matters: Visualizations can be misleading. Statistical tests tell you whether a pattern is real or could be due to random chance.

Test 1: Chi-Square Test for Independence

  • Question: Is there a statistically significant relationship between Contract type and Churn?
  • Method: scipy.stats.chi2_contingency()
  • How:
  1. Create a contingency table: pd.crosstab(df['Contract'], df['Churn'])
  2. Pass it to chi2_contingency()
  3. Get: chi2, p_value, dof, expected
  • Interpret: If p_value < 0.05, reject the null hypothesis → Contract type significantly affects churn

Test 2: Independent T-Test

  • Question: Do churned customers pay significantly different MonthlyCharges?
  • Method: scipy.stats.ttest_ind()
  • How:
  1. Split the data: group_churned = df[df['Churn']==1]['MonthlyCharges']
  2. And: group_stayed = df[df['Churn']==0]['MonthlyCharges']
  3. Run: ttest_ind(group_churned, group_stayed)
  • Interpret: If p_value < 0.05, the difference in monthly charges is statistically significant

Report Format (print this for each test):

Test: Chi-Square — Contract vs. Churn
Chi2 Statistic: ____
P-Value: ____
Degrees of Freedom: ____
Conclusion: [Reject / Fail to reject] the null hypothesis.
Interpretation: [Your sentence]

Phase 6 — Feature Engineering & Preprocessing (1 hour)

This phase prepares data for machine learning. Follow the exact order below to avoid data leakage.

Step 1 — Encode categorical variables:

  • For binary Yes/No columns (e.g., PhoneService, PaperlessBilling): use .map({'Yes': 1, 'No': 0})
  • For multi-class columns (e.g., Contract, InternetService): use pd.get_dummies(df, drop_first=True)
  • After encoding, ALL columns in X must be numeric (integer or float)

Step 2 — Separate features and target:

  • X = all columns except Churn
  • y = the Churn column

Step 3 — Train/Test Split:

  • 80% training, 20% testing
  • Use stratify=y to preserve the class ratio in both sets
  • Use random_state=42 for reproducibility

Step 4 — Handle class imbalance with SMOTE:

  • Import SMOTE from imblearn.over_sampling
  • Apply it ONLY to training data: X_train_res, y_train_res = smote.fit_resample(X_train, y_train)
  • NEVER apply SMOTE to the test set — test set must reflect real-world distribution
  • Check the new class distribution in y_train_res — should be balanced

Step 5 — Scale numerical features:

  • Use StandardScaler on numerical columns: tenure, MonthlyCharges, TotalCharges
  • Fit ONLY on X_train_res, then transform both X_train_res and X_test
  • If you fit on the test set, you have data leakage — this inflates your results artificially

Phase 7 — Model Training & Evaluation (2 hours)

Model 1: Logistic Regression (Baseline)

  • Simple, interpretable, fast
  • Use max_iter=1000 to ensure convergence
  • This is your baseline — other models should beat it

Model 2: Random Forest Classifier

  • More powerful, non-linear model
  • Start with n_estimators=100

For each model, produce ALL of the following:

  1. Predictions on test set: y_pred = model.predict(X_test)
  2. Classification Report: classification_report(y_test, y_pred) — shows precision, recall, F1
  3. Confusion Matrix: Plot as a heatmap with sns.heatmap()
  4. AUC-ROC Score: roc_auc_score(y_test, model.predict_proba(X_test)[:,1])
  5. ROC Curve: Plot both models on the same chart for comparison
  6. 5-Fold Cross-Validation: cross_val_score(model, X, y, cv=5, scoring='roc_auc')

Important — Metric Selection:

  • Do NOT use accuracy alone. With imbalanced data, a model that always predicts "No Churn" gets 73% accuracy but is useless.
  • Use F1-score (especially for class 1 = churned) and AUC-ROC as primary metrics.
