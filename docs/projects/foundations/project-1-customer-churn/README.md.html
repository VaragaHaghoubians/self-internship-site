Project 1 — Telecom Customer Churn Prediction

Role: Data Scientist Difficulty: Intermediate Estimated Time: 8–12 hours

Problem Statement

You are a Data Scientist at a telecommunications company. The business has noticed a rising rate of customers canceling their subscriptions (churning). The cost of acquiring a new customer is 5–7x higher than retaining an existing one — so predicting and preventing churn is a high-priority business problem.

Your mission:

  • Understand who churns and why through exploratory data analysis
  • Validate your observations with statistical hypothesis tests
  • Build a classification model to predict which customers are likely to churn
  • Deliver business recommendations based on your findings

This is a complete, end-to-end data science workflow.

Dataset

Name: Telco Customer Churn Source: Kaggle Direct Link: https://www.kaggle.com/datasets/blastchar/telco-customer-churn File: WA_Fn-UseC_-Telco-Customer-Churn.csv Size: ~7,000 rows, 21 columns

How to Download:

  1. Create a free account at https://www.kaggle.com (if you don't have one)
  2. Go to the dataset link above
  3. Click the Download button
  4. Rename the file to telco_churn.csv
  5. Place it inside a data/ folder in this project directory

Dataset Columns (key ones):

ColumnTypeDescription
customerIDStringUnique identifier (not useful for modeling)
genderCategoricalMale / Female

Skills Demonstrated

  • Exploratory Data Analysis (EDA) — univariate, bivariate, multivariate
  • Statistical hypothesis testing — Chi-Square test, independent t-test
  • Data preprocessing — encoding, scaling, handling class imbalance
  • Classification modeling — Logistic Regression, Random Forest
  • Model evaluation — F1-score, AUC-ROC, confusion matrix, cross-validation
  • Feature importance analysis
  • Business communication and data storytelling

Expected Outputs

By the end of this project you will have produced:

  1. 7+ visualizations saved as PNG files in outputs/figures/
  2. 2 statistical test results with written interpretations
  3. 2 trained models with full evaluation reports
  4. An ROC curve comparison between models
  5. A feature importance chart from the Random Forest
  6. A written business summary with actionable recommendations

Key Concepts to Understand First

Before coding, make sure you understand:

  • What is class imbalance and why it matters
  • The difference between precision and recall — when does each matter?
  • What the AUC-ROC score represents
  • Why you should never apply SMOTE or scaling to the test set
  • The difference between Logistic Regression and Random Forest
