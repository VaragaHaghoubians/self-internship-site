Prerequisites — Project 2: Retail Sales Performance Analysis

Role: Data Analyst | Phase: 1 | Suggested timing: After Month 1–2

This project has no machine learning. It is pure data analysis — loading, cleaning, aggregating, visualising, and interpreting business data. Every skill here is foundational and will be used in every project that follows.

pandas — Everything You Need

This project is 90% pandas. Master these operations before starting:

Loading & inspecting

import pandas as pd

df = pd.read_csv("superstore.csv", encoding="latin-1")  # note: may need latin-1 encoding
df.head()
df.info()
df.describe()
df.isnull().sum()          # find missing values
df.dtypes                  # check column types

Datetime handling

# Parse date strings into actual datetime objects
df["Order Date"] = pd.to_datetime(df["Order Date"])
df["Year"]  = df["Order Date"].dt.year
df["Month"] = df["Order Date"].dt.month
df["YearMonth"] = df["Order Date"].dt.to_period("M")  # groups by month

Critical: If you skip date parsing, all time-trend analysis is impossible.

Groupby — the most-used operation

# Total sales by category
df.groupby("Category")["Sales"].sum()

# Profit by region AND category
df.groupby(["Region", "Category"])["Profit"].sum().reset_index()

# Multiple aggregations at once
df.groupby("Segment").agg(
    total_sales=("Sales", "sum"),
    avg_discount=("Discount", "mean"),
    total_profit=("Profit", "sum"),
)

Sorting & ranking

df.sort_values("Profit", ascending=False).head(10)           # top 10 most profitable orders
df.groupby("Sub-Category")["Profit"].sum().sort_values()     # worst to best sub-categories

Derived columns

df["Profit Margin"] = df["Profit"] / df["Sales"]            # margin as a ratio
df["Is Profitable"]  = df["Profit"] > 0                     # boolean flag

Pivot tables

# Heatmap-ready format: regions as rows, categories as columns, profit as values
pivot = df.pivot_table(values="Profit", index="Region", columns="Category", aggfunc="sum")

Business Metrics You Must Understand

This project asks you to answer business questions. Before starting, understand what each metric means in plain English — not just how to compute it.

MetricFormulaBusiness meaning
Profit marginProfit / SalesFor every $1 sold, how many cents are kept as profit?
Discount impactScatter: Discount vs. ProfitHigh discounts often destroy profit — the dataset shows this clearly
Year-over-year growth(this_year - last_year) / last_yearIs the business growing or shrinking?
Average order valuetotal_sales / num_ordersHow much does a typical order bring in?

Key insight to find: In the Superstore dataset, the Furniture category is often the most problematic — high sales but low/negative profit. You should discover this through analysis, not just state it.

Analytical Thinking — The Skill That Separates Analysts

Knowing pandas is not enough. You need to know what question to ask next. Practise this loop:

  1. Look at the data. Form a hypothesis. ("I think high discounts cause negative profit.")
  2. Write the code to test it. (df.groupby("Discount_bucket")["Profit"].mean())
  3. Interpret the result. ("Yes — orders with >40% discount are on average unprofitable.")
  4. Frame it as a business recommendation. ("Recommend capping discounts at 30% for Furniture.")

A Data Analyst who cannot frame findings as recommendations is just a query machine. Practice the recommendation step.
