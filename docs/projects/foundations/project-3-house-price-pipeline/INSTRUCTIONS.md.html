Phase 4 — Feature Engineering (2 hours)

This is where ML Engineering separates from basic modeling.

Step 4.1 — Handle Missing Values (understand the WHY):

For this dataset, many "missing" values are NOT missing — they mean the feature doesn't exist. For example, a missing PoolQC means "no pool", not that we don't know.

Rules:

  • For categorical columns where NaN means "None/Not applicable":

→ Fill with the string 'None' or 'No' → Example: PoolQC, Alley, Fence, GarageType, BsmtQual

  • For numerical columns where NaN means 0:

→ Fill with 0 → Example: GarageArea, GarageCars, BsmtFinSF1, BsmtFinSF2

  • For truly missing values (e.g., LotFrontage):

→ Fill with median, grouped by neighborhood:

  df['LotFrontage'] = df.groupby('Neighborhood')['LotFrontage'].transform(
      lambda x: x.fillna(x.median())
  )

This domain-specific imputation is a key ML engineering skill.

Good features make models work much better than good algorithms.

Step 4.3 — Encode Ordinal Features:

Some features have a natural order that should be preserved:

ExterQual: Po < Fa < TA < Gd < Ex  →  1, 2, 3, 4, 5
BsmtQual:  None < Po < Fa < TA < Gd < Ex  →  0, 1, 2, 3, 4, 5

Create a mapping dictionary for these quality columns and apply them with .map(). This is better than one-hot encoding because the order matters.

Step 4.4 — Identify and Remove Outliers:

  • Remove houses where GrLivArea > 4000 and SalePrice < 200000
  • These are statistical outliers that will distort your model
  • Only 2–4 rows — negligible data loss

Step 4.5 — Log-Transform the Target:

  • y = np.log1p(df['SalePrice'])
  • np.log1p is log(1 + x) — handles the case where SalePrice could be 0
  • When making final predictions, you must reverse this:

final_price = np.expm1(log_prediction)

Phase 5 — Build the sklearn Pipeline (2 hours)

This is the most important part of the ML Engineering project.

What a Pipeline does: It chains preprocessing steps and a model into a single object. When you call pipeline.fit(X_train, y_train), it:

  1. Fits the preprocessors on X_train
  2. Transforms X_train
  3. Fits the model on the transformed data

When you call pipeline.predict(X_test), it:

  1. Transforms X_test using the already-fitted preprocessors
  2. Returns predictions

This guarantees NO data leakage and makes deployment trivial.

Step 5.1 — Define column lists:

numerical_cols = [...]    # list of all numerical feature columns
categorical_cols = [...]  # list of all categorical/nominal columns

Step 5.2 — Define transformers:

# For numerical columns:
# - SimpleImputer(strategy='median')  → fill remaining NaN with median
# - StandardScaler()                  → scale to mean=0, std=1

# For categorical columns:
# - SimpleImputer(strategy='most_frequent')  → fill remaining NaN with mode
# - OneHotEncoder(handle_unknown='ignore', sparse_output=False)
#   → encode as binary columns; handle_unknown='ignore' is CRITICAL
#   → if a new category appears at prediction time, it doesn't crash

Step 5.3 — Build ColumnTransformer:

# preprocessor = ColumnTransformer(transformers=[
#     ('num', numerical_pipeline, numerical_cols),
#     ('cat', categorical_pipeline, categorical_cols)
# ])

Step 5.4 — Add model to Pipeline:

# full_pipeline = Pipeline(steps=[
#     ('preprocessor', preprocessor),
#     ('model', YourChosenModel())
# ])

Step 5.5 — Fit and evaluate:

  • Use cross_val_score with scoring='neg_root_mean_squared_error'
  • Negate the result: RMSE = -scores.mean()
  • Remember: predictions are in log scale, so RMSE is also in log scale
  • A good RMSE on this dataset is around 0.11–0.14

Phase 7 — Hyperparameter Tuning (1 hour)

For your best model (likely XGBoost or Ridge), tune hyperparameters:

For XGBoost:

param_grid = {
    'model__n_estimators': [300, 500, 700],
    'model__learning_rate': [0.05, 0.1],
    'model__max_depth': [3, 4, 5],
    'model__subsample': [0.8, 1.0]
}

Note: Parameters in a Pipeline are prefixed with the step name and __. So n_estimators inside the 'model' step becomes 'model__n_estimators'.

Use GridSearchCV with cv=5, scoring='neg_root_mean_squared_error'.

Phase 8 — Model Persistence (30 minutes)

Save your trained pipeline so it can be loaded without retraining:

joblib.dump(best_pipeline, 'outputs/models/house_price_pipeline.pkl')

Test that it loads and predicts correctly:

Phase 9 — Residual Analysis (45 minutes)

Residuals = actual values - predicted values. A good model has random, normally-distributed residuals. Patterns in residuals indicate that the model is missing something.

Plots to create:

  1. Residuals vs. Predicted Values:
  • X-axis: predicted values
  • Y-axis: residuals (actual - predicted)
  • Should look like random scatter around y=0
  • If you see a curve or funnel shape, there is a problem
  1. Histogram of Residuals:
  • Should be approximately bell-shaped (normal distribution)
  • If heavily skewed, the model is systematically over/under-predicting
  1. Q-Q Plot of Residuals:
  • scipy.stats.probplot(residuals, plot=plt)
  • Points should fall along the diagonal line
  • Deviations at the ends indicate non-normality
