Project 3 — House Price Prediction: End-to-End ML Pipeline

Role: ML Engineer Difficulty: Intermediate–Advanced Estimated Time: 10–14 hours

Problem Statement

You are a Machine Learning Engineer at a real estate analytics company. Unlike a Data Scientist (who focuses on exploration and insights), your job as an ML Engineer is to build a robust, reproducible, production-grade machine learning pipeline that can be retrained on new data and deployed to serve predictions.

The business problem: given a set of house features (size, neighborhood, year built, quality rating, etc.), predict the sale price of a house accurately.

Your engineering responsibilities:

  • Build a reusable scikit-learn Pipeline that handles preprocessing automatically
  • Apply rigorous feature engineering to extract value from raw data
  • Compare multiple regression models and select the best one
  • Track experiments and evaluate models properly
  • Serialize (save) your trained pipeline so it can be loaded and used without retraining
  • Document your work so another engineer can reproduce it

Dataset

Name: House Prices — Advanced Regression Techniques Source: Kaggle Competition Direct Link: https://www.kaggle.com/competitions/house-prices-advanced-regression-techniques/data Files: train.csv, test.csv, data_description.txt Size: 1,460 training samples, 80 features

How to Download:

  1. Go to the Kaggle competition link above
  2. You may need to accept the competition rules to access the data
  3. Click Data tab → Download train.csv and test.csv
  4. ALSO download data_description.txt — READ IT before starting. It explains every column.
  5. Place all files in the data/ folder of this project

Why This Dataset?

This is one of the most famous Kaggle datasets for regression. It has:

  • 80 features (numerical + categorical)
  • Many missing values to handle
  • Significant feature engineering opportunities
  • A real competition leaderboard for benchmarking

Key ML Engineering Concepts

This project focuses on engineering skills, not just modeling:

ConceptWhat It Means
sklearn PipelineChains preprocessing + model into one object — prevents data leakage
ColumnTransformerApplies different transformers to different column types
Custom TransformerA class that implements fit() and transform() for custom logic
Cross-ValidationEvaluates model on multiple folds — more reliable than single split
Target TransformationLog-transforming SalePrice to reduce skewness — improves model performance
Model SerializationSaving the pipeline to disk with joblib.dump() so it can be deployed
RMSE vs. MAERoot Mean Squared Error penalizes large errors more than MAE

Evaluation Metric

The competition uses Root Mean Squared Log Error (RMSLE):

$$\text{RMSLE} = \sqrt{\frac{1}{n}\sum_{i=1}^{n}(\log(p_i + 1) - \log(a_i + 1))^2}$$

In practice: log-transform SalePrice first, then use regular RMSE. This penalizes underestimation more than overestimation.
