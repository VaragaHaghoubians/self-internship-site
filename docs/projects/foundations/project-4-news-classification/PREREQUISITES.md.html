Part 1 — Classical NLP: TF-IDF + Logistic Regression

What TF-IDF is (understand this conceptually before coding it)

TF (Term Frequency): How often does the word "touchdown" appear in this article? IDF (Inverse Document Frequency): How many articles contain the word "touchdown"? If it appears in most articles, it is not informative — IDF penalises common words. TF-IDF score: High for words that appear often in this article but rarely in others → these are the discriminating words.

from sklearn.feature_extraction.text import TfidfVectorizer
from sklearn.linear_model import LogisticRegression
from sklearn.pipeline import Pipeline
from sklearn.metrics import classification_report

# Build the classical NLP pipeline
tfidf_pipeline = Pipeline([
    ("tfidf", TfidfVectorizer(
        max_features=50_000,      # keep the 50k most informative words
        ngram_range=(1, 2),       # include single words AND two-word phrases
        sublinear_tf=True,        # apply log(1+tf) — compresses large counts
        stop_words="english",     # remove "the", "is", "at", etc.
    )),
    ("clf", LogisticRegression(
        max_iter=1000,
        C=5.0,                    # inverse regularisation strength
        multi_class="multinomial",
    )),
])

tfidf_pipeline.fit(train_texts, train_labels)
preds = tfidf_pipeline.predict(test_texts)
print(classification_report(test_labels, preds, target_names=["World", "Sports", "Business", "Sci/Tech"]))

Text preprocessing steps to understand

import re

def clean_text(text: str) -> str:
    text = text.lower()                         # lowercase everything
    text = re.sub(r"[^a-z\s]", "", text)       # remove punctuation and numbers
    text = re.sub(r"\s+", " ", text).strip()   # collapse whitespace
    return text

Part 2 — Deep Learning: BERT Transformer

Conceptual understanding required before writing code

You must be able to answer these questions before touching the Hugging Face code:

  1. What is a token?

BERT does not read words — it reads tokens. The word "playing" might become ["play", "##ing"]. Numbers split into digits. This is why tokenizer.encode("playing") might return 3 token IDs, not 1.

  1. What is an embedding?

Each token is converted into a high-dimensional vector (768 numbers for BERT-base). Words with similar meanings end up as similar vectors. This is what allows BERT to understand that "automobile" and "car" mean the same thing without being told explicitly.

  1. What is an attention mechanism?

When reading the word "bank" in "river bank", the transformer looks at all other words in the sentence simultaneously and decides that "river" is relevant context — so it adjusts the embedding for "bank" accordingly. This is why transformers outperform older sequence models (LSTM/RNN).

  1. What does fine-tuning mean?

BERT was pre-trained on billions of words from Wikipedia and books. Fine-tuning means you take that pre-trained model and train it a little more on your specific task (news classification) — updating the weights slightly to specialise for your data. This is vastly more efficient than training from scratch.

Hugging Face basics

from transformers import AutoTokenizer, AutoModelForSequenceClassification, Trainer, TrainingArguments
from datasets import Dataset
import torch

# 1. Load the pre-trained tokenizer and model
model_name = "bert-base-uncased"
tokenizer  = AutoTokenizer.from_pretrained(model_name)
model      = AutoModelForSequenceClassification.from_pretrained(model_name, num_labels=4)

# 2. Tokenise your text
def tokenize(batch):
    return tokenizer(batch["text"], truncation=True, padding="max_length", max_length=128)

# 3. Convert pandas DataFrames to Hugging Face Datasets
train_dataset = Dataset.from_pandas(train_df[["text", "label"]])
test_dataset  = Dataset.from_pandas(test_df[["text", "label"]])

train_dataset = train_dataset.map(tokenize, batched=True)
test_dataset  = test_dataset.map(tokenize, batched=True)

# 4. Define training configuration
training_args = TrainingArguments(
    output_dir="./results",
    num_train_epochs=3,
    per_device_train_batch_size=16,
    per_device_eval_batch_size=32,
    evaluation_strategy="epoch",
    save_strategy="epoch",
    load_best_model_at_end=True,
    fp16=torch.cuda.is_available(),   # use GPU mixed precision if available
)

# 5. Train
trainer = Trainer(
    model=model,
    args=training_args,
    train_dataset=train_dataset,
    eval_dataset=test_dataset,
)
trainer.train()

Evaluation Metrics

Use macro F1-score as the primary metric — not accuracy. The four classes in AG News are balanced, but macro F1 is the standard for multi-class NLP problems.

from sklearn.metrics import f1_score, classification_report

f1_macro = f1_score(y_true, y_pred, average="macro")
print(f"Macro F1: {f1_macro:.4f}")

# Full breakdown per class:
print(classification_report(y_true, y_pred, target_names=["World", "Sports", "Business", "Sci/Tech"]))

Benchmark targets:

  • TF-IDF + Logistic Regression: ~91% accuracy
  • Fine-tuned BERT: ~94–95% accuracy
  • The 3–4% gap is the cost of the 10–100x longer training time — understand this trade-off.

GPU vs. CPU — Practical Notes

BERT fine-tuning on CPU takes many hours. If you do not have a GPU:

  • Use Google Colab (free T4 GPU — go to Runtime → Change runtime type → GPU)
  • Use a smaller model: "distilbert-base-uncased" — 40% fewer parameters, 60% faster, ~1% less accurate
  • Reduce max_length from 512 to 128 — most news headlines + first sentence fit in 128 tokens
