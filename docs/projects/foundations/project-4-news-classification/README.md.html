Project 4 — News Article Classification with NLP & Transformers

Role: AI Engineer Difficulty: Advanced Estimated Time: 12–18 hours

Problem Statement

You are an AI Engineer at a digital news aggregation company. The platform receives thousands of news articles every day and needs to automatically categorize them into topics (Sports, Technology, Business, World News, etc.) so they can be routed to the correct section of the website.

Manually labeling articles is slow and expensive. Your job is to build an AI-powered text classification system using state-of-the-art Natural Language Processing (NLP) techniques. Specifically:

  1. Build a classical NLP baseline using TF-IDF + Logistic Regression
  2. Build an advanced model using BERT (a pre-trained Transformer) fine-tuned on the news data
  3. Evaluate both approaches and compare them
  4. Understand how the model makes predictions using attention visualization

This project represents the cutting edge of what companies deploy in production AI systems today.

Dataset

Name: AG News Classification Dataset Source: Hugging Face Datasets (easiest) or Kaggle Size: 120,000 training articles, 7,600 test articles, 4 categories

Option A — Hugging Face (Recommended, no download needed):

from datasets import load_dataset
dataset = load_dataset("ag_news")

Install first: pip install datasets

Option B — Kaggle Download:

Link: https://www.kaggle.com/datasets/amananandrai/ag-news-classification-dataset Files: train.csv, test.csv Place in data/ folder.

Two-Part Approach

Part 1 — Classical NLP (TF-IDF Baseline):

  • Text cleaning and preprocessing
  • TF-IDF vectorization
  • Logistic Regression / SVM classifier
  • Evaluation: accuracy, F1-score, confusion matrix

Part 2 — Transformer Fine-Tuning (BERT):

  • Load pre-trained bert-base-uncased from HuggingFace
  • Tokenize data with BertTokenizer
  • Fine-tune the model on AG News for 3 epochs
  • Evaluate and compare with baseline

Skills Demonstrated

  • Text preprocessing (cleaning, tokenization, stopwords, stemming/lemmatization)
  • TF-IDF vectorization and N-gram features
  • Training classical NLP classifiers
  • Loading and using HuggingFace pre-trained models
  • Tokenizing text for Transformer models
  • Fine-tuning BERT with a classification head
  • Visualizing attention weights
  • Comparing classical vs. deep learning NLP approaches
  • Understanding what BERT "sees" in text

Key Concepts to Understand

ConceptWhat It Is
TF-IDFTerm Frequency-Inverse Document Frequency — numerical representation of text
BERTBidirectional Encoder Representations from Transformers — pre-trained on massive text
TokenizationBreaking text into tokens (sub-words) that the model understands
Fine-tuningTaking a pre-trained model and training it further on your specific task
AttentionMechanism that lets BERT focus on relevant words when classifying
Transfer LearningReusing knowledge from a model trained on a large dataset
