Cloud coding mentor

Use this website for guidance, then use a cloud IDE for real projects.

The browser lessons are for practice. For full projects, use GitHub Codespaces first. It gives you VS Code, a terminal, Git, Python, Node, and project files in one browser workspace.

A

Recommended path: GitHub Codespaces

Use this when you want the easiest beginner setup with a real terminal and code editor in the browser.

  1. Create a GitHub account.
  2. Create a repository named self-internship.
  3. Upload this whole folder to the repository.
  4. Open the repository, choose Code, then Codespaces, then create a codespace.
  5. In the terminal, run the month command from this website.
  6. Commit your work after every task so your progress is saved.

Reality checks

The hard parts are expected.

Month 3 and Month 5 will feel slow.

Docker and Django force you to think like a production engineer. If they take longer than four weeks, keep the scope small and finish the deliverable.

Apply before the roadmap is finished.

After Month 2 you can apply for analyst roles. After Month 4 you can apply for junior ML roles. After Month 6 you can apply for ML engineer or backend Python roles.

Keep hardware boring.

Use one ESP32, one DHT11 sensor, one endpoint, and one chart. The value is the data pipeline, not the number of wires.

Daily habit

Save your work to GitHub every single day.

Run these 4 commands in VS Code terminal every time you finish a lesson, task, or project part. This is how your work is backed up and your portfolio grows.

The 4 commands

1
git status

See what files changed before saving

2
git add .

Select all changed files

3
git commit -m "Month 1 Day 2: practiced Python loops"

Save a snapshot on your computer

4
git push

Send the snapshot to GitHub

Remember: Ctrl+S saves file → git commit saves in Git → git push saves on GitHub
